Relative Attributes: Implementation and Performance Analysis

Learning Visual Attribute Rankings from Pairwise Image Comparisons
Computer Vision Project | 2025

What are Relative Attributes?

  • Visual attributes as mid-level semantic representations
  • Relative attributes: "Person A is more smiling than Person B"
  • vs Binary attributes: "Is person smiling? Yes/No"
  • Why relative? More natural supervision, richer vocabulary, better generalization

Problem Formulation

  • Goal: Learn ranking function that assigns real-valued scores
  • Higher scores indicate stronger attribute presence
  • For attribute a: learn fa(x) where fa(xi) > fa(xj)
  • Key advantages: Captures continuous spectrum, enables zero-shot learning

Datasets Used

  • Outdoor Scene Recognition (OSR):
  • 2,688 images across 8 scene categories
  • 6 visual attributes to predict
  • Category order example: tallbuilding < insidecity < ... < forest
  • Labeled Faces in the Wild (LFW): 80 identities for zero-shot learning

Feature Representation

  • GIST Descriptors (512-dim):
  • Global scene structure and spatial layout
  • Gabor Filter Banks (32 filters):
  • 4 scales × 8 orientations
  • Texture and orientation information
  • Lab Color Histograms (45-dim):
  • Luminance and color information
  • Total: 557-dimensional feature vector

Ranking SVM Method

  • Learn linear ranking function fa(x) = wT x for each attribute
  • Optimization problem: Maximize margin while satisfying ordering constraints
  • Ordered pairs Ω: Images with clear attribute ordering
  • Similar pairs Σ: Images with approximately equal attribute strength
  • Solved via quadratic programming (CVXOPT)

Baseline Method: Binary SVM

  • Convert continuous orderings to binary labels:
  • 1. Map categories to levels 0-7
  • 2. Threshold at midpoint: level ≥ 4 → label 1, else → label 0
  • 3. Train LinearSVC on binary labels
  • Evaluate by using classifier scores for pairwise ranking

Attribute-Specific Performance

  • Largest improvements:
  • 'open': +470.4% (16.98% → 96.86%)
  • 'natural': +158.6% (36.48% → 94.34%)
  • 'close-depth': +143.1% (40.88% → 99.37%)
  • Best absolute performance: 'perspective' & 'close-depth' (99.37%)
  • Smallest gain: 'large-objects' (+10.6%) - already easier to classify

Why Ranking Outperforms Binary

  • Binary classification limitations:
  • Loss of granularity from continuous orderings
  • Arbitrary threshold may misalign with true boundary
  • Inconsistent labels for nearby categories
  • Ranking SVM advantages:
  • Direct pairwise modeling optimized for ranking
  • Continuous scores capture full attribute spectrum
  • Margin maximization for better generalization

Feature Impact Analysis

  • GIST Descriptors: Essential for 'natural', 'open', 'perspective'
  • Gabor Filters: Critical for 'diagonal-plane' and object attributes
  • Color Information: Differentiates natural vs man-made scenes
  • Combined 557-dim vector provides:
  • Rich semantic representation
  • Computational efficiency
  • Complementary information sources

Zero-Shot Learning Extension

  • Describe unseen categories via relative attribute comparisons
  • Three approaches compared:
  • DAP: Binary attribute classifiers
  • SRA: Ranking with generative modeling
  • Proposed: Full ranking-based method
  • Evaluation: Varying attribute counts from k=1 to k=6
  • Result: Learned rankers generalize effectively to unseen categories

Limitations & Future Directions

  • Current limitations:
  • Hand-crafted features vs deep learning (ResNet, VGG)
  • Linear model may miss nonlinear relationships
  • Quadratic programming expensive for large datasets
  • Future improvements:
  • Deep ranking networks with end-to-end learning
  • Multi-task learning across correlated attributes
  • Attention mechanisms for attribute-relevant regions

Real-World Applications

  • Fine-grained recognition: Distinguish bird species, clothing styles
  • Image retrieval: "Show images more formal than this"
  • Zero-shot learning: Recognize new categories without training
  • E-Commerce: Product recommendations by relative attributes
  • Medical imaging: Comparative diagnosis (privacy-preserving)
  • Fashion: Compare products for recommendation systems

Conclusions

  • Full zero-shot learning implementation

  • Impact: Enables fine-grained attribute reasoning and zero-shot learning
